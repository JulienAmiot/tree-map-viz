§17.104 — ComputedCard + ComputedBusinessScoreCard Lit visual layer

First UI strand for the round-7 Computed* node kinds. Lands two new Lit
components at src/adapters/ui/views/ComputedNode/ComputedCards.ts:

  - `<computed-card>` for `ComputedNode<T>` tiles
  - `<computed-business-score-card>` for `ComputedBusinessScoreNode<T>` tiles

Both render a Σ aggregation badge + the auto-derived value (numeric or
empty-reason) + a `<select>` kind dropdown that dispatches a
`bubbles+composed` `computation-kind-change` event with
`{ nodeId, newKind }`. The CBSC variant additionally renders the BSC-style
objective row + corner timestamp + trend arrow inherited from the
`BusinessScoreNode` objective surface.

Extends `NodeKind` from 2 to 4 inhabitants (+`ComputedNode`,
+`ComputedBusinessScoreNode`), adds `ComputationKindName` Lit-friendly
mirror of the §17.95 enum names, adds `ComputedValueViewModel`
(`numeric|empty` discriminated union), adds `ComputedNodeViewModel` +
`ComputedBusinessScoreNodeViewModel`, registers both tags in
`nodeViewRegistry` for both asParent + asChild roles (computed tiles
collapse the role distinction — value-area identical in both).

Renderers are inlined rather than reused from §17.40's `valueTemplate`
because that module's helpers are typed against the v3
`BusinessScoreCardNodeViewModel` (different value shape); re-typing them
polymorphically would touch v3 code beyond §17.104's scope.

`ComputedNode<T>` and lands `ComputedBusinessScoreNode<T>` in the
`RangedValueNode` branch as `BusinessScoreCardNode` VMs — both behaviours
unchanged from pre-§17.104. Mapper extension is a follow-on strand
(§17.110 cutover or a dedicated wiring strand).
